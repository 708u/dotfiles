# セルフエディティング

初期ドラフトを改善するための反復的な編集プロセス。

## 基本テクニック

### スタイルガイドの採用

一貫性と品質を確保するためにスタイルガイドに従う。

重要な原則:

- 能動態の使用
- 段階的手順は番号付きリスト
- その他のリストは箇条書き
- 用語の一貫性

### 読者視点の検討

ドラフトを対象読者の視点から読み直す:

- ドキュメントの目的は明確か
- 読者にとって不慣れな用語には定義があるか
- 前提知識は適切に記載されているか
- 読者が次に何をすべきか明確か

### 声に出して読む

不自然な表現や長すぎる文を特定するため、原稿を音声で確認する。

効果:

- 文のリズムの問題を発見
- 冗長な表現を特定
- 読みにくい箇所を把握

スクリーンリーダーの使用も効果的。

### 時間を置いて見直す

初期ドラフト後、時間を置いて新鮮な目で再検討する。

```txt
推奨: 最低でも数時間、可能であれば1日以上
```

時間を置くことで、ほぼ常に改善点が見つかる。

### 文脈を変える

異なる環境で確認することで改善点が見つかりやすくなる:

| 方法 | 効果 |
|------|------|
| 印刷版で確認 | 画面では見落としやすい問題を発見 |
| フォントを変更 | 新鮮な目で読める |
| サイズを変更 | レイアウトの問題を発見 |
| 別のデバイスで確認 | レスポンシブの問題を発見 |

## ピアエディターの活用

### 効果的なレビューの依頼

技術専門家でなくても、以下の観点でフィードバックは有益:

- スタイルガイドへの準拠
- 文章の明確さ
- 論理の流れ
- 用語の一貫性

### フィードバックの受け取り方

- 防御的にならない
- 具体的な改善提案を求める
- すべてのフィードバックを検討（採用は選択的でよい）

## 編集チェックリスト

### 構造

- [ ] 導入部でスコープと対象読者を明示
- [ ] 見出しは内容を正確に反映
- [ ] 情報は論理的な順序で配置
- [ ] ナビゲーションは適切

### 内容

- [ ] 各段落は1つのトピックに集中
- [ ] 冒頭文は段落の内容を要約
- [ ] 例は適切で理解を助ける
- [ ] 不要な情報は削除

### スタイル

- [ ] 能動態を使用
- [ ] 文は簡潔（1文1アイデア）
- [ ] 用語は一貫
- [ ] 代名詞の参照は明確

### 形式

- [ ] リストは並列構造
- [ ] 表は整形されている
- [ ] コードブロックは正しくフォーマット
- [ ] リンクは有効

## 反復的改善プロセス

```txt
1. 初期ドラフト作成
2. 時間を置く
3. 声に出して読む
4. 構造を確認
5. 内容を精査
6. スタイルを統一
7. ピアレビューを依頼
8. フィードバックを反映
9. 最終確認
```

各段階で1つの観点に集中することで、効率的に品質を向上できる。
