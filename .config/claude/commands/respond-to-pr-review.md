# PR Review Response Workflow

PRレビューコメントへの対応を体系的に進めるワークフロー。

PR番号: $ARGUMENTS

## コンテキスト収集

ワークフロー開始前に以下の情報を収集する:

- 現在のブランチ
- PR情報（タイトル、本文、コミット履歴）
- レビューコメント一覧

## ワークフロー

### レビューコメント取得

PRのレビューコメントを取得し、対応が必要なものを特定する。

GitHub MCPを使用してコメントを取得:

```txt
mcp__github__get_pull_request_comments
mcp__github__get_pull_request_reviews
```

コメントを以下のカテゴリに分類:

- **コード修正が必要**: 実装変更を伴う指摘
- **回答のみ**: 質問や確認事項への回答
- **対応不要**: メモ、承認コメント等

### 対応計画作成

計画ファイルに以下を記載:

- 対象ファイル一覧
- 各コメントへの対応方針
  - 修正内容の具体的な記述
  - 回答のみの場合はその内容
- 対応の優先順位

### コード修正

計画に従い修正を実施:

- 関連する修正は1つのコミットでまとめる
- コミットメッセージにはレビュー対応であることを明記

### テスト実行

修正箇所に関連するテストを実行し、全てパスすることを確認。

### コミット・プッシュ

修正をコミットしてプッシュ。

### PR Description更新確認

プッシュ後、PR descriptionが最新の状態を反映しているか確認する。

以下の場合はdescriptionを更新:

- 中間コミットで追加した内容がdescriptionに反映されていない
- レビュー対応で機能や仕様が変更された
- 検証項目やテストカバレッジに変更がある

更新にはGitHub MCPを使用:

```txt
mcp__github__update_pull_request
```

### コメント返信

各レビューコメントに対してGitHub MCPで返信:

```txt
mcp__github__add_pull_request_review_comment (インラインの場合)
mcp__github__add_issue_comment (一般コメントの場合)
```

返信フォーマット:

```txt
[claude code] 対応しました: {commit_hash}

{変更内容の簡潔な説明}
```

質問への回答の場合:

```txt
[claude code] {回答内容}
```

## 注意事項

- 複数のレビュアーがいる場合は、レビュアーごとにコメントをグループ化
- 既に削除されたコードへのコメントには、削除された旨を回答
- 対応が複数コミットにまたがる場合は、関連するコミットハッシュを全て記載
